Vanguard UI Design Guidelines

Web Application & Website Specification • v1.1 • Developer & AI Reference

Single authoritative UI/UX design system for all Vanguard web applications and websites. Desktop-first, responsive, token-driven, accessible.

1. IT Principles

All UI decisions must trace back to these five core principles:

Principle Meaning UI Implication
User-FirstReduce cognitive loadNext action obvious
Autonomy by DesignEnable self-serviceNo IT dependency
Process-FirstReflect real workflowsOrder → Execute → Post
IntegratedConsistent patternsSame components everywhere
Speed with ControlFast & predictableSafe, reversible actions

2. Scope

In Scope

Web applications, public websites, internal tools, admin interfaces, AI-assisted UX.

Out of Scope

Native mobile apps, ERP implementation details.

Approach

Desktop-first, responsive tablet/mobile, grid-based, 8pt spacing system.

Primary Actions (Accent Colour)

Multiple Primary Actions

6.1. Page Layout & Hierarchy (WMS)

Define the overarching page structure used in warehouse management scenarios. Use this hierarchy to organise information clearly and support efficient workflows.

Hierarchical Structure

  • Header: Displays the current warehouse, mode (e.g. Manual / Automatic) and user account information. Uses a neutral background with brand-blue text for context.
  • Primary Menu: Shows the active module (e.g. Outbound) alongside global search, primary settings and user role icons. Actions sit on the same row as the module label to reduce height.
  • KPI Row: Includes a small set of metric cards summarising key indicators (e.g. Outstanding Orders, Lines to Pick). Cards use radius-sm (4 px) and neutral backgrounds for a crisp, compact appearance.
  • Secondary Menu: Provides page-level actions. Primary actions (e.g. Create fulfillment) align left; distinct or contextual actions (e.g. Update ERP) align right to separate them visually.
  • Table Menu: Integrated contextual search appears on the left with a clear label (e.g. “Search fulfillments”). Table-specific actions like column visibility and export live on the far right.
  • Table: Dense tables with sortable/filterable columns. Support multi-select checkboxes, bulk action toolbars and row actions on hover. Align text left and numbers right. Use minimal row heights and radius-sm corners for a utilitarian look.

Design Notes

  • Minimal Colour: Rely on neutral shades for most surfaces. Reserve accent and semantic colours for statuses, tags and interactive states.
  • Compact Radii: Use the smallest radius token (radius-sm = 4 px) on cards, buttons and tables to mirror the crisp, industrial feel of WMS tools.
  • Spacing: Maintain tight vertical spacing between each hierarchical row (8–16 px) to preserve vertical real estate while keeping elements distinct.
  • Responsiveness: The hierarchy compresses gracefully on smaller screens, collapsing menu rows into dropdowns or stacked panels while preserving order.

8. AI & Data-Driven UX

AI is assistive, explainable, and accountable.

AI RECOMMENDATION

Optimise Zone B routing for pedestrian safety.

Based on last 7 days traffic data. Assumes standard operating hours.

AI UX Principles

  • Assistive, Not Authoritative: AI suggests; humans decide
  • Explain Before Trust: Show data and assumptions
  • Reversible: Allow override without penalty

9. Accessibility (WCAG AA)

Contrast

4.5:1 for text, 3:1 for UI. Colour never sole indicator.

Keyboard

All elements reachable. Logical tab order. No trapped focus.

Focus

Always visible. Returns logically after modal close.

Semantics

Native HTML. ARIA where needed. Labels linked.
