Vanguard UI Design Guidelines

Customer Portal Specification • v2.0 • Developer & AI Reference

Single authoritative UI/UX design system for Vanguard customer portals. Desktop-first, responsive, token-driven, accessible.

1. IT Principles

All UI decisions must trace back to these five core principles:

Principle Meaning UI Implication
User-FirstReduce cognitive loadNext action obvious
Autonomy by DesignEnable self-serviceNo support dependency
Process-FirstReflect real workflowsBrowse → Select → Action
IntegratedConsistent patternsSame components everywhere
Speed with ControlFast & predictableSafe, reversible actions

2. Approach

Desktop-First, Responsive

Design for desktop as the primary experience. Use columnar layouts and card-based panels that gracefully reflow on tablet and mobile. Desktop is authoritative; mobile prioritises key tasks over density.

Grid-Based Layout

Use CSS Grid and Flexbox for layouts. Standard patterns include 4-column metric grids, 3-column dashboards, and master-detail panels. All spacing uses multiples of 8px.

Neutral-Dominant Palette

80–90% neutral colours for the interface. Reserve brand-blue (#004674) and brand-red (#ff0000) for identity accents only—logos, branded headers, and occasional emphasis. Use the accent colour (#0891B2) for primary actions and interactive states.

Customer Portal Focus

Design patterns optimised for customer-facing scenarios: account summaries, order history, invoices, support tickets, and profile management. Dashboards highlight account health and pending actions rather than operational metrics.

7. Global Navigation

Horizontal module bar for primary navigation. Icons with labels, clear active states.

Vanguard
Dashboard Orders Invoices Support

Rules

  • Icons + labels: Navigation items must have both icon and text label
  • Active state: Use accent colour background (accent-light) and text (accent)
  • Max items: 5–7 primary items visible; overflow into "More" dropdown
  • Responsive: Collapse to hamburger menu on mobile

Horizontal vs Vertical Navigation

  • Horizontal (top bar): Customer portals, marketing sites, simpler apps with ≤7 modules
  • Vertical (side bar): Complex admin interfaces, apps with many modules, when deep hierarchy needed

Rules

  • Keyboard navigation: Arrow keys to navigate, Enter to select, Escape to close
  • Grouped results: Organise by entity type (Orders, Invoices, Support Tickets, etc.)
  • Match highlighting: Use accent colour background for matched text
  • Focus management: First result focused by default; trap focus within modal

9. Tabs, Nested Tabs & Segmented Controls

Primary Tabs

Switch between content sections (different content).

Nested Tabs

Subdivide details within a section. Visually lighter than primary tabs.

Nested tab content appears here...

Segmented Control

Toggle view modes (same content, different presentation). Max 5 options, equal widths.

When to Use What

  • Primary Tabs: Different content sections (Overview / Billing / Messages)
  • Nested Tabs: Subdivisions within a section (Activity / Invoices / Tickets)
  • Segmented Control: View modes for same data (List / Grid), time periods (Day / Week / Month)

13. Tables

Primary interface for customer records. Integrated search, multi-select, bulk actions, and view toggle.

Order Date Status Amount
ORD-2024-0128 Jan 10, 2024 In Transit $1,450.00

Table Rules

  • Search: Integrated search with meaningful default sorting
  • Multi-select: Checkboxes reveal bulk actions when items selected
  • Row actions: Appear on hover to reduce clutter
  • View toggle: Switch between table and card views
  • Alignment: Left-align text, right-align numbers

16. Settings & Configuration Modals

Large modals with vertical tabs for multi-section settings. Consistent Save/Cancel buttons.

Account Settings

Profile Information

Modal Rules

  • Vertical tabs: Use for 3+ settings sections
  • Focus management: Trap focus within modal, return focus on close
  • Scrollable content: Content area scrolls, header/footer fixed
  • Consistent actions: Cancel (left), Save (right, primary)
  • Escape to close: Always support keyboard dismissal

17. Interactive Panels & Toolbars

For timelines, plan views, or interactive content. Vertical toolbar on left, contextual actions at bottom.

Account Activity Timeline

Toolbar Rules

  • Touch targets: 44×44px minimum for all tools
  • Active state: Use accent colour background
  • Vertical placement: Left side for primary tools
  • Context menu: Bottom bar for secondary actions and status

19. Accessibility (WCAG AA)

Contrast

4.5:1 for text, 3:1 for UI. Colour never sole indicator.

Keyboard

All interactive elements reachable. Escape closes modals.

Focus

Always visible. Trap in modals. Return on close.

Touch Targets

44×44px minimum for all interactive elements.
